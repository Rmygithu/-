# 多轮对话使用指南

## 概述

Breakup Recovery Squad 现在支持**真正的多轮长上下文对话**，你可以：

- 与每个 Agent 进行持续的深度交流
- 页面刷新后对话记忆不丢失
- 追问、修改、深入探讨任何话题

## 技术实现

### Agno 原生记忆

- 每个 Agent 使用独立的 `session_id` 实现对话隔离
- 使用 `add_history_to_context=True` 自动管理上下文
- `num_history_messages=20` 保留最近 20 条消息（约 10 轮对话）

### 滑动窗口保护

- 超过 10 轮对话时，自动保留最近 10 轮
- 避免上下文过长导致模型响应变慢
- 重要信息在窗口滑动时可能会丢失，建议及时保存

## 使用场景

### 1. 一键生成后的追问

```
用户：我和男朋友分手了，很难过...

[一键生成]

疗愈师：我理解你的感受，分手确实很痛苦...

用户：他是因为我太粘人分手的，我是不是做错了？

疗愈师：（基于之前的对话记忆，理解"他"指的是前男友）
你愿意聊聊具体是什么让你觉得自己"太粘人"吗？
有时候我们以为的"粘人"，其实只是正常的陪伴需求。
```

### 2. 信件修改迭代

```
用户：帮我写一封不会寄出的信

释怀专家：好的，这里是开头...

用户：语气太愤怒了，能改成平静一点的吗？

释怀专家：（记得之前的信件内容）
好的，我调整一下...

用户：还是有点太正式，能不能更口语化？

释怀专家：（继续修改，不重复之前的版本）
这样呢？
```

### 3. 计划调整

```
用户：给我一个恢复计划

规划师：第一天，建议你...

用户：这个太难了，我连床都不想起

规划师：（理解用户的状态）
那我们从更小的一步开始：
今天只需要做一件事——拉开窗帘，让阳光进来。
你觉得这个能做到吗？
```

### 4. 深度探讨

```
用户：分析一下我们为什么会分手

分析师：从你的描述来看...

用户：但我觉得主要问题不在这里

分析师：（尊重用户的观点）
你说得对，我可能只看到了表面。
你觉得真正的问题是什么？

用户：其实是我们沟通方式不一样

分析师：（深入探讨沟通问题）
这是一个很重要的观察。
能举一个具体的例子吗？
我想了解你们的沟通模式是怎么出问题的。
```

## Agent 引导风格

每个 Agent 的引导风格不同：

| Agent | 引导风格 | 示例问题 |
|-------|----------|----------|
| 🤗 疗愈师 | 温和倾听 | 「你现在最想聊的是什么？」 |
| ✍️ 释怀专家 | 温暖陪伴 | 「你想对他说什么？」 |
| 📅 规划师 | 实用鼓励 | 「明天你打算先从哪个开始？」 |
| 💪 分析师 | 深度探讨 | 「你觉得这个分析有道理吗？」 |

## 注意事项

1. **对话记忆是自动的**：无需手动管理，Agent 会自动记住之前的对话

2. **页面刷新不会丢失**：对话记忆存储在 Agno 的 session 中，刷新页面后可继续

3. **滑动窗口**：超过 10 轮后，最早的内容会逐渐淡出

4. **每个 Tab 独立**：4 个 Agent 的对话是完全独立的

5. **清空对话**：点击「清空全部对话」会同时清空所有 Agent 的记忆

## 常见问题

### Q: 为什么 Agent 不记得之前的对话？

A: 可能的原因：
- 你清空了对话历史
- 超过了滑动窗口大小（10 轮）
- 页面刷新后 session_id 发生了变化（理论上不会）

### Q: 能否增加对话记忆的轮数？

A: 可以修改 `agents/base.py` 中的 `MAX_HISTORY_ROUNDS` 值，但不建议超过 20 轮，可能影响响应速度。

### Q: 一键生成的内容可以继续追问吗？

A: 可以！一键生成的内容会自动写入 Agent 记忆，你可以在此基础上继续追问、修改、深入探讨。
